### 功能模块概述
该功能模块旨在为NPML翻译器CLI工具增加一个新参数 `-p`，用于主动从线上同步更新NPML提示词。此功能允许用户在不等待缓存过期的情况下，强制获取最新的提示词规范。

### 开发类型
修改

### 可能涉及到的代码文件
1.  `main.ts`: CLI工具的入口和参数解析逻辑。
2.  `prompt_cache.ts`: 管理NPML提示词缓存的核心逻辑。

### 可能用到的标准库、框架、第三方库
*   **Deno标准库**:
    *   `https://deno.land/std@0.208.0/flags/mod.ts`: 用于解析命令行参数 (已在 `main.ts` 中导入)。
    *   `jsr:@std/path` 和 `jsr:@std/fs`: 用于处理文件路径和文件系统操作 (已在 `prompt_cache.ts` 中导入)。
*   **内置API**:
    *   `Deno.args`: 获取命令行参数。
    *   `fetch`: 用于从线上获取提示词文件内容。

### 功能的具体实现细节
1.  **参数解析**: 在 `main.ts` 中，修改现有的参数解析逻辑 (`Deno.args`)，为 `parse` 函数添加一个新的布尔类型参数 `-p` (或 `--prompt-sync`)。
2.  **逻辑分支**: 在 `main` 函数中，添加一个条件判断，检查是否传递了 `-p` 参数。如果传递了，则执行同步逻辑，完成后直接退出程序（不进行后续的NPML请求文生成）。
3.  **同步逻辑**: 在 `prompt_cache.ts` 中，`PromptCache` 类需要提供一个方法（例如 `syncPrompt`），该方法接受一个URL（可选，从配置文件或默认值获取），强制执行网络请求获取最新内容，并覆盖本地缓存文件，无论缓存是否有效。
4.  **集成**: 在 `main.ts` 的同步逻辑分支中，实例化或获取 `PromptCache` 对象，并调用其同步方法。
5.  **帮助信息**: 更新 `printHelp` 函数，将新参数 `-p` 添加到帮助信息中。

### 可能会创建的类、接口和函数
*   **函数 (main.ts)**: 无新函数，修改现有的 `main` 函数和 `printHelp` 函数。
*   **函数 (prompt_cache.ts)**: 在 `PromptCache` 类中增加一个 `syncPrompt(url?: string): Promise<void>` 方法。

### 与项目其他功能的关系（依赖关系）
*   **依赖**: 此功能依赖于 `prompt_cache.ts` 中的 `PromptCache` 类及其缓存路径、网络获取逻辑。
*   **影响**: 此功能是CLI工具的一个独立选项，不影响其他模块的核心逻辑。它会与现有的缓存机制交互，强制更新缓存文件。
